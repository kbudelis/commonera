<decisions_made>

- The latest mobile flow and Blue Zodiac/light-paper direction supersede the PRD's original sequence and visual specifications.
- The six states are welcome -> zodiac transition -> optional `MM/DD/YYYY` birthday -> conditional personal reveal -> current-month reading -> moon/Friday upcoming.
- Welcome and birthday both offer **Skip to this month**. Skipping removes the personal section rather than leaving an empty frame.
- Sections live on one continuous vertical page. Use soft proximity settling, not mandatory hard snap; never leave the user trapped between frames.
- Use `easeInOutCubic` for visual and programmatic transitions. Do not override native touch-scroll physics.
- Reduced motion keeps every state and all navigation while removing zoom, rotation, parallax, and auto-scroll.
- Blue Zodiac progresses from full-screen welcome to full-width birthday state to top-edge arc. The post-welcome background matches the source image's light paper color.
- v1 is deterministic and client-only: bundled typed data, no runtime AI, database, login, or server.
- Current-month content is rich; the other eleven birth profiles may use lighter templates.
- The personal layer combines current-month guidance with a birth-month reflection; it is not a planetary-transit claim.
- Moon content is a clearly symbolic Hebrew-calendar state. Exact astronomy is deferred.
- v1 is screenshot-ready only. Custom share export, full 12-month browsing, rich copy for all months, and the production rotary zodiac are deferred.
</decisions_made>

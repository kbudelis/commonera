## Trust Boundaries

| Boundary | Description |
|---|---|
| visitor input → client flow reducer | A user-provided date string affects only local placeholder navigation; it must be validated before entering the personal branch. |
| browser preference → motion behavior | Reduced-motion preference must alter presentation only, never remove a reachable state or control. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Tampering | birthday input / `transitionFlow` | low | mitigate | Validate the complete MM/DD/YYYY civil date before emitting the birthday-success action; keep invalid input on the birthday state. |
| T-01-02 | Denial of service | mobile scroll and focused input | medium | mitigate | Preserve document scrolling, use proximity-only settling, avoid overflow locks and scroll interception, and test input focus at 390 × 844. |
| T-01-03 | Information disclosure | client application | low | accept | Phase 1 stores no birthday data, account, network request, or server response; no sensitive data crosses the browser boundary. |
| T-01-04 | Denial of service | `scrollToLandmarkUnlessReducedMotion` | low | mitigate | Query the browser preference inside the exported App-level helper and return before every scroll API call when reduced motion is enabled; cover the guard with Node assertions. |
</threat_model>

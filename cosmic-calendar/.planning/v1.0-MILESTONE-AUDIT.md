The GSD integration check ran through a generic-agent workaround because the
typed `gsd-integration-checker` role was unavailable in this runtime. Its
evidence was then re-evaluated against the reconciled contract.

2. Birthday input → Hebrew facts → normalized month → authored portrait
3. `MONTH_ENTRIES` → month gallery selection → reading/ritual/grounding display
4. Current Hebrew date → Friday/moon derivation → upcoming cards
5. Stored civil input/name → defensive rehydration → populated edit/re-entry

### Accepted presentation changes, not blockers

- Birthday owns the skip; the reducer's welcome-origin skip branch is retained
  only as a harmless historical test path.
- Name is required only for the personal route; skip remains zero-input.
- Derived Hebrew date/moon/tribe facts are not all rendered as a data table; the
  current portrait presentation is the accepted Phase 03.1 contract.
- The month identity is represented by zodiac label, correct Hebrew letter art,
  selected constellation, and accessible option label; the exact derived Hebrew
  month label remains in the model.
- Full-screen post-birth settlement intentionally supersedes the Phase 1
  proximity-only placeholder behavior.

# Digital Shabbat — build kernel

This folder is both the build system's context and the handoff
documentation. Reading order:

1. `CLAUDE.md` — the constitution: product, laws, tone, team protocol
2. `research/meetings/` — the opening-meeting brief: the client's own frame,
   sprint deliverables, and evaluation criteria. Primary reference for the
   overall project.
3. `contracts/` — pinned: tokens, state/interfaces, canonical copy
4. `lanes/` — five parallel workstreams (five Claude Code instances)
5. `research/` — the aesthetic + ritual reasoning behind the decisions
6. `chronicle/CHRONICLE.md` — the decision record of the build itself
7. `sync/` — live coordination: per-lane status + herald's digest

Built at the Common Era × Vibe Coding Sprint, July 10–11 2026, by Mike
Marrotte (Signalform) and a small team of agents.
